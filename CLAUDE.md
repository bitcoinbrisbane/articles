# Articles Repository

## Purpose
Critical, well-sourced articles about cryptocurrency, Web3, and blockchain technology for LinkedIn publication.

## Writing Guidelines

- **Always cite claims.** Every factual claim must link to a primary source (whitepaper, official docs, on-chain data, SEC filings, reputable reporting). No unsourced assertions.
- Use inline citations with markdown links, e.g. `[source text](url)`.
- Include a **References** section at the bottom of every article listing all sources.
- Quote directly from primary sources (whitepapers, EIPs, BIPs, official docs) where possible.
- **Voice: first-person, personal.** Open with a personal hook — what were *you* promised? What did *you* experience? Lead with the human angle, then back it up with data. Examples: "I was promised AWS without AWS", "I tried to run a local node and here's what happened."
- Tone: opinionated but fair. Present facts first, then analysis.
- Structure: use numbered sections with clear headings.
- End with a call-to-action inviting discussion.
- Articles are markdown files in the repo root.
- Update `README.md` table when adding a new article.

## Source Hierarchy
1. Primary sources (whitepapers, protocol specs, on-chain data, SEC filings)
2. Reputable crypto journalism (CoinDesk, The Block, Decrypt, TechCrunch)
3. Research reports (Messari, CoinGecko Research)
4. Project official blogs and documentation
